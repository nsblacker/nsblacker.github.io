工作一年之后的技术规划

发表于 2016-11-06   |     |   阅读次数

前言

入职一年了，准确地说是技术储备将近一年了，期间主要从事iOS的开发工作，从存在Safari的书签基本可以看出在iOS开发道路上已经走的路线图，唐巧前辈提供的技术路线我基本上都跟着走了一遍。与此同时，不忘接触学习了一些经典或者比较前沿的技术，比如：

  1. 用hexo+github.ioda搭建博客系统；
  2. 顺便学着用MarkDown写写文档；
  3. 看看Github的原理；
  4. NativeActive开发模式；
  5. WebRTC涉及到的服务器；
  6. Java实现个爬虫（这个我还是比较满意的，终于实现了一次模拟登陆）
  7. CartoGrapher、SLAM
  8. Google Project Tango（AR device）
  9. 微信小程序开发
    …
    其实大多数只是开了个头，有些甚至只把首页存到了书签中，深思中～

后面的工作安排

贪多不行！浅尝辄止不行！
按照过去一年来的个人时间分布，对之后的成长路线有了一个模糊的打算。主要以公司工作作为主线，在其分支中附加一些安排。

  • iOS

    • 以OC为主要语言，将架构思维带入到日常编码中，趁着项目二期的时间投入，本着低耦合 高复用 易拓展 易维护的原则，重构之前的代码。
    • UIKit中提供的控件，如UITableView UISearchBar 等独立性比较强的控件，借鉴已有开源项目实现最好的编码架构。
    • 重构时，使用OmniGraffle做UML图与Class图
    • 进阶性的知识（runtime，block，gcd那种）
    • Swift，不学
    • 自己的项目，不做

  • 机器学习

    • 不学

  • JavaScript

    • 不学

  • 微信小程序

    • 不学

  • NativeActive

    • 不学

总结

我的意思是什么呢？
我的意思是我根本他妈的不想多学这些后来接触的东西[二哈]
我还要练尤克里里啊，我还要烘培啊，我还要健身啊，我还要读政经史文啊，我还要玩密室逃脱啊，我不仅要玩，我还得玩到可以秀出来。为自己贴上更多标签啊！
所以，下面这些才是我的安排

  • 尤克里里 —— 你得玩下去才行
  • 烘焙 —— 社交网络靠这个，不过是买个烤箱而已
  • 政经史文 —— 自卑时读书就好了
  • 健身 —— 不是光减肥啊，胸上腹上的那一点点肥肉，减下去才能看见块啊
  • 密室逃脱 —— 要不是奔着雁哥

要不是奔着雁哥～
